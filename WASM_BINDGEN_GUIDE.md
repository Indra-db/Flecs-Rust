# WASM-Bindgen with WASI-libc: Two-Stage Build Guide

This guide explains how to use **wasm-bindgen** with **wasi-libc** in Rust projects, solving the fundamental compatibility issue between these two technologies.

## The Problem

**wasm-bindgen** and **wasi-libc** are incompatible because:

1. The `wasm32-unknown-unknown` Rust target automatically includes WASI runtime functions (like `__wasilibc_initialize_environ`)
2. wasm-bindgen's interpreter cannot understand these complex WASI functions during code generation
3. This causes errors like: `__wasilibc_initialize_environ: unknown instruction Block(...)`

## The Solution: Two-Stage Build

Our approach separates the "binding generation phase" from the "runtime execution phase":

1. **Stage 1**: Generate JavaScript bindings from a minimal WASM file (no WASI dependencies)
2. **Stage 2**: Build the full runtime WASM file (with WASI/wasi-libc support)
3. **Stage 3**: Replace the runtime WASM file while keeping the JavaScript bindings

This allows us to have both wasm-bindgen JavaScript interop AND wasi-libc C library support.

## Project Structure

```
your_project/
├── Cargo.toml              # Main project with Flecs + WASI
├── src/
│   ├── lib.rs              # Full implementation with Flecs ECS
│   └── bindgen.rs          # wasm-bindgen exports
├── bindgen_minimal/        # Minimal project for binding generation
│   ├── Cargo.toml          # Standalone, no workspace, minimal deps
│   └── src/
│       └── lib.rs          # Stub functions matching your exports
├── web/
│   └── test.html           # Test page using the bindings
├── pkg/                    # Generated by wasm-bindgen (auto-created)
└── Makefile.toml           # Cargo-make tasks
```

## Step-by-Step Implementation

### 1. Create the Minimal Binding Generator

Create `bindgen_minimal/Cargo.toml`:

```toml
[package]
name = "your_project_bindgen_minimal"
version = "0.1.0"
edition = "2021"

[workspace]
# This package is standalone, not part of the workspace

[lib]
crate-type = ["cdylib"]

[dependencies]
wasm-bindgen = "0.2"
js-sys = "0.3"

[dependencies.web-sys]
version = "0.3"
features = ["console"]

[profile.release]
opt-level = "s"
lto = true
codegen-units = 1
panic = "abort"
```

Create `bindgen_minimal/src/lib.rs` with **exact same function signatures** as your main project:

```rust
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
extern "C" {
    #[wasm_bindgen(js_namespace = console)]
    fn log(s: &str);
}

macro_rules! console_log {
    ($($t:tt)*) => (log(&format_args!($($t)*).to_string()))
}

// Export the EXACT same functions as your real implementation
#[wasm_bindgen]
pub fn your_main_function() {
    console_log!("Stub - will be replaced with real implementation");
}

#[wasm_bindgen] 
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

// Add all other exported functions with matching signatures
```

### 2. Set Up Cargo-Make Tasks

Add to your `Makefile.toml`:

```toml
[tasks.build-bindgen]
description = "Build with wasm-bindgen using two-stage approach"
script = [
  'echo "🦀 Two-stage WASM-bindgen build process"',
  'echo "Stage 1: Generate JS bindings from minimal WASM (no WASI)..."',
  '(cd bindgen_minimal && cargo build --target wasm32-unknown-unknown --release)',
  '(cd bindgen_minimal && wasm-bindgen --target web --no-typescript --out-dir ../pkg target/wasm32-unknown-unknown/release/your_project_bindgen_minimal.wasm)',
  'echo "✅ Generated JS bindings successfully"',
  'echo "Stage 2: Build full runtime WASM (with WASI)..."',
  'cargo build --target wasm32-unknown-unknown --no-default-features --features "your_features"',
  'echo "Stage 3: Replace WASM file with runtime version..."',
  'cp ../../target/wasm32-unknown-unknown/debug/your_project.wasm ./pkg/your_project_bindgen_minimal_bg.wasm',
  'echo "✅ Two-stage build complete!"',
]
workspace = false

[tasks.serve-bindgen]
description = "Build and serve the wasm-bindgen demo"
dependencies = ["build-bindgen"]
script = [
  'echo "🦀 Your Project WASM-Bindgen Demo"',
  'echo "✅ JS bindings from minimal WASM (no WASI conflicts)"',
  'echo "✅ Runtime uses full implementation (with WASI support)"',
  'echo "🌐 Open http://localhost:8000/web/test.html"',
  'python3 -m http.server 8000'
]
workspace = false

[tasks.test-bindgen]
description = "Verify the two-stage build"
dependencies = ["build-bindgen"]
script = [
  'echo "🧪 Verifying two-stage build..."',
  'ls -la pkg/*.js pkg/*.wasm',
  'wasm2wat pkg/*_bg.wasm | grep -c wasilibc || echo "WASI functions: $?"',
  'echo "✅ Verification complete"',
]
workspace = false
```

### 3. Create Test HTML Page

Create `web/test.html`:

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <title>Your Project WASM-Bindgen Test</title>
</head>
<body>
    <h1>Your Project with WASM-Bindgen</h1>
    <button onclick="testFunction()">Test Main Function</button>
    <div id="output"></div>

    <script type="module">
        import init, { your_main_function, add } from './pkg/your_project_bindgen_minimal.js';
        
        async function initWasm() {
            try {
                // Provide WASI stubs if needed
                const wasiImports = {
                    wasi_snapshot_preview1: {
                        clock_time_get: (id, precision, time_ptr) => {
                            const timeNs = BigInt(Math.floor(performance.now() * 1000000));
                            const memory = new DataView(wasmInstance.exports.memory.buffer);
                            memory.setBigUint64(time_ptr, timeNs, true);
                            return 0;
                        },
                        proc_exit: () => 0,
                        environ_get: () => 0,
                        environ_sizes_get: () => 0,
                        fd_write: () => 0,
                        fd_close: () => 0,
                        random_get: () => 0,
                        // Add other WASI functions as needed
                    }
                };
                
                await init(undefined, wasiImports);
                
                window.your_main_function = your_main_function;
                window.add = add;
                
                document.getElementById('output').innerHTML = 'WASM initialized successfully!';
            } catch (error) {
                document.getElementById('output').innerHTML = `Error: ${error}`;
                console.error('WASM init error:', error);
            }
        }
        
        window.testFunction = function() {
            window.your_main_function();
        };
        
        initWasm();
    </script>
</body>
</html>
```

## How It Works

### Stage 1: Binding Generation
- Builds `bindgen_minimal` → produces clean WASM without WASI functions
- `wasm-bindgen` processes this clean WASM → generates JavaScript bindings
- No interpreter errors because no complex WASI instructions

### Stage 2: Runtime Build  
- Builds main project → produces full WASM with WASI/wasi-libc support
- Includes all your actual functionality, Flecs ECS, etc.

### Stage 3: File Replacement
- Replaces the minimal WASM file with the full runtime WASM
- JavaScript bindings remain unchanged and compatible
- Runtime gets full WASI functionality

## Key Benefits

✅ **wasm-bindgen compatibility** - No interpreter errors  
✅ **Full WASI support** - Use wasi-libc, file I/O, environment variables  
✅ **Complete functionality** - Your full Rust/C implementation  
✅ **JavaScript interop** - Proper TypeScript bindings, web integration  
✅ **Cross-platform** - Works with any WASI-compatible runtime  

## Troubleshooting

### Build Issues
- Ensure `bindgen_minimal` has exact same function signatures
- Check that `bindgen_minimal` is outside the workspace (has `[workspace]` table)
- Verify wasm-bindgen CLI is installed: `wasm-bindgen --version`

### Runtime Issues  
- Add missing WASI function stubs to JavaScript imports
- Check browser dev console for import errors
- Verify WASM file was properly replaced in pkg/ directory

### Function Signature Mismatches
- Keep `bindgen_minimal/src/lib.rs` in sync with your main exports
- Use same parameter types and return types
- Test basic functions first (like `add`) before complex ones

## Advanced Tips

1. **Automate signature sync**: Use a script to extract `#[wasm_bindgen]` functions from main project
2. **Optimize builds**: Use `--release` for minimal project, `--debug` for main project during development  
3. **CI/CD**: Run `cargo make test-bindgen` in your pipeline to verify builds
4. **Documentation**: Generate TypeScript definitions with `wasm-bindgen --typescript`

This approach successfully bridges the gap between wasm-bindgen's JavaScript ecosystem and wasi-libc's C compatibility, giving you the best of both worlds.
